MyGarage
1. Общее описание
Создать веб-приложение на Flask для учета автомобилей в личном
владении. Доступ к данным должен быть строго ограничен: каждый
пользователь видит только те машины, которые он сам добавил в
систему.

2. Функциональные требования
А. Система пользователей (Auth)
• Регистрация:
◦ Поля: Имя пользователя (username), Пароль (password).
◦ Валидация: Имя пользователя должно быть уникальным.
• Авторизация:
◦ Вход по имени пользователя и паролю.
◦ Выход из системы (Logout).

Б. Управление гаражом (CRUD)
• Просмотр: Главная страница со списком всех добавленных
автомобилей пользователя.
• Добавление: Форма создания новой записи.
◦ Поля: Марка (н-р, BMW), Модель (н-р, M5), Год выпуска, Тип
топлива (бензин/дизель/электро).
• Удаление: Возможность убрать машину из гаража (например,
если она продана).

3. Структура базы данных (Модели)
Понадобятся две таблицы со связью «Один ко многим»
1. User (Пользователь): id, username, password_hash.
2. Car (Автомобиль): id, brand, model, year, fuel_type, user_id
(внешний ключ, привязывающий машину к владельцу).

4. Интерфейс (Страницы)
• index.html — Приветственная страница (Landing).
• register.html — Форма регистрации.
• login.html — Форма входа.
• dashboard.html — Личный кабинет со списком машин и кнопкой
«Добавить».
• add_car.html — Форма добавления авто.

5. Этапы разработки (План действий)
1. Настройка окружения: Установка Flask, Flask-SQLAlchemy и
Flask-Login.
2. Создание моделей: Описание структуры базы данных в коде.
3. Настройка Auth: Написание логики регистрации и входа.
4. Маршруты (Routes): Создание функций, которые открывают
нужные страницы.
5. Шаблоны: Написание HTML-кода для отображения данных
