Flask. Это отличный выбор для старта в веб-разработке на Python — он легкий, понятный и не навязывает лишних правил.

Для начала давай создадим твое первое приложение («Hello World»).

Шаг 1. Установка

Тебе понадобится сам Flask. Лучше устанавливать его в виртуальное окружение:

bash

pip install flask

Use code with caution.

Шаг 2. Минимальный код

Создай файл app.py и вставь туда этот код:

python

from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello_world():
    return 'Привет! Это твое первое приложение на Flask.'

if __name__ == '__main__':
    app.run(debug=True)

Use code with caution.

Шаг 3. Запуск

Запусти файл через терминал:

bash

python app.py

Use code with caution.

После этого переходи в браузере по адресу http://127.0.0.1.

    Функция

    url_for нужна для того, чтобы не прописывать пути к страницам вручную. Вместо того чтобы везде писать /user/Jhon, ты просишь Flask: «Найди маршрут для функции profile и подставь туда имя Jhon».

    Давай разберем твой пример по строчкам:

    1. url_for('index') → Вывод: /
      Самый простой случай. Ты передаешь имя функции (index), а Flask возвращает путь, который указан у нее в @app.route.
    2. url_for('login') → Вывод: /login
      Аналогично — получаем прямой путь к странице логина.
    3. url_for('login', next='/') → Вывод: /login?next=%2F
      Вот тут интересно. Если ты передаешь в url_for аргумент, которого нет в описании маршрута (в @app.route('/login') нет переменных), Flask автоматически добавляет его в конец как GET-параметр (после знака ?). Это часто используют, чтобы запомнить, куда вернуть пользователя после входа.
    4. url_for('profile', username='Jhon Doe') → Вывод: /user/Jhon%20Doe
      А здесь аргумент username есть в самом маршруте @app.route('/user/<username>'). Поэтому Flask не добавляет его в конец, а вставляет прямо в середину пути. Пробел автоматически заменяется на %20 (безопасный формат для ссылок).

    Зачем это использовать вместо обычных строк?

    • Гибкость: Если ты решишь изменить адрес с /login на /auth/signin, тебе не придется искать и менять ссылку по всему проекту. Ты поменяешь её один раз в @app.route, а url_for('login') сам сгенерирует новый путь.
    • Безопасность: Она сама кодирует спецсимволы (как пробел выше).

      интернет-магазин.

      У нас есть список товаров и страница конкретного товара. Если мы будем писать ссылки вручную, и вдруг решим изменить структуру сайта (например, переименовать папку в адресе), нам придется переписывать сотни ссылок. url_for решает эту проблему.

      Смотри на этот код:

      python

      from flask import Flask, url_for

app = Flask(__name__)

# Маршрут для главной страницы
@app.route('/shop/items/')
def catalog():
    return "Это каталог товаров"

# Маршрут для страницы товара (принимает ID товара)
@app.route('/shop/item/<int:item_id>')
def show_item(item_id):
    return f"Вы смотрите товар под номером {item_id}"

with app.test_request_context():
    # 1. Просто получаем ссылку на каталог
    print(url_for('catalog')) 
    # Выведет: /shop/items/

    # 2. Генерируем ссылку для товара с ID 42
    # Flask видит переменную <item_id> в маршруте и подставляет её туда
    print(url_for('show_item', item_id=42)) 
    # Выведет: /shop/item/42

    # 3. А что если мы хотим добавить параметры поиска (фильтры)?
    # Переменной 'color' нет в маршруте catalog, поэтому Flask добавит её через '?'
    print(url_for('catalog', color='red', size='XL'))
    # Выведет: /shop/items/?color=red&size=XL

      Use code with caution.

      В чем главная «фишка»?

      Представь, что завтра ты решил, что адрес /shop/items/ звучит некрасиво и поменял его на /market/products/.

      Коротко:

        Вот как это работает на практике. Главный секрет в том, что

        url_for привязывается к имени функции (def catalog), а не к тексту в кавычках внутри @app.route.

        Шаг 1: Старый адрес

        Допустим, у тебя был такой код:

        python

        @app.route('/shop/items/') # Старый адрес
def catalog():
    return "Каталог"

# В коде (или в шаблоне) ты вызываешь:
print(url_for('catalog')) 
# Результат: /shop/items/

        Use code with caution.

        Шаг 2: Ты меняешь адрес (Ребрендинг!)

        Теперь ты решил, что адрес должен быть /market/products/. Ты меняешь только одну строчку в настройках маршрута:

        python

        @app.route('/market/products/') # НОВЫЙ адрес
def catalog():
    return "Каталог"

# Тот же самый вызов в коде:
print(url_for('catalog')) 
# Результат АВТОМАТИЧЕСКИ стал: /market/products/

        Use code with caution.

        В чем магия?

        Если бы ты везде в приложении (в кнопках, в меню, в письмах пользователям) писал путь вручную как строку "/shop/items/", то при смене адреса тебе пришлось бы делать "Поиск и замену" по всему проекту. И если бы ты забыл хоть одну ссылку, она бы выдала ошибку 404.

        С url_for('catalog') тебе всё равно, какой там сейчас адрес. Ты просто говоришь: "Эй, Flask, дай мне актуальную ссылку на функцию catalog".
