сначала создадим основу проекта: 
django-admin startproject locallibrary
cd ./locallibrary/

locallibrary/
    manage.py
    locallibrary/
        settings.py
        urls.py
        wsgi.py

Подпапка проекта locallibrary c именем locallibrary это ключевая директория нашего
    проекта.

*settings.py содержить все настройки проекта. Здесь мы регистрируем приложения,
    задаем размещение статчиных файлов, настройки баз данных и т.д.
*urls.py задает асоциации url адресов с представлениями. несмотря на то что этот файл 
    может содержать все настройки url, обычно его делят на части, по одной на приложение.
*wsgi.py используется для налаживания связи между вашим Django приложением и сервером.
manage.py используется для создания поиложений, работы с базами данных и для 
    запуска отладочного сервера.

далее нам будет нужно создать приложение проекта.
    python3 manage.py startapp catalog # мы сделали приложение
    на уровне с manage.py появится папка с файлами определяющими приложение.
регистрируем приложение ./locallibrary/settings.py
    В INSTALLED_APPS дописываем приложение catalog следующим образом
    INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'catalog.apps.CatalogConfig',
]
        
Откройте locallibrary/locallibrary/urls.py и обратите внимание на 
    закомментированный текст, который объясняет суть происходящего.
    Сейчас нам нужно добавить путь catalog в urlpatterns как показано ниже фразы:
    Including another URLconf.
    urlpatterns = [
        path('admin/', admin.site.urls),
        path('catalog/', include('catalog.urls')),
    ]
Теперь перенаправим корневой url на url/catalog/
